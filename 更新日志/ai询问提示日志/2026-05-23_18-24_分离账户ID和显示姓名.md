# AI 询问提示日志：分离账户ID和显示姓名

**日期：** 2026-05-23 18:24

## 用户反馈

用户希望修改登录系统：
1. 登录界面使用"账户ID"和"密码"登录
2. 修改个人信息时，"姓名"仅作为显示名称，不影响登录
3. 增添修改账户ID和密码的功能

## AI 分析过程

1. 检查后端 User 模型：当前 `username` 字段同时用于登录和显示
2. 检查后端 auth.py：登录使用 `username` 查询用户
3. 检查后端 user.py：`update_profile` 可修改 `username`，会影响登录
4. 检查前端 login.html：登录表单使用"用户名"
5. 检查前端 profile.html：EditProfileModal 修改"用户名"

**关键发现**：需要将 `username` 字段的两个职责分离

## 结论

需要实现以下改动：
1. 后端 User 模型添加 `account_id` 字段用于登录
2. 保留 `username` 字段仅作为显示名称
3. 修改登录/注册 API 使用 `account_id`
4. 添加修改账户ID和密码的 API
5. 修改前端表单和弹窗

## 实现方案

1. 后端添加 `account_id` 字段（unique, not null）
2. 创建数据库迁移脚本，将现有 `username` 复制到 `account_id`
3. 修改登录/注册 API 使用 `account_id`
4. 添加 `PUT /user/account-id` 和 `PUT /user/password` 接口
5. 前端登录表单改为"账户ID"，注册表单添加"账户ID"和"姓名"
6. 新增 AccountIdModal 和 PasswordModal 弹窗

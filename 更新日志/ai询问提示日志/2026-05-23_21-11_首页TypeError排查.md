# AI 询问提示日志：首页 TypeError 排查

**日期：** 2026-05-23 21:11

## 用户反馈

用户报告运行时控制台出现错误：
```
TypeError: Cannot read properties of undefined (reading 'toLocaleString')
at initPage (http://localhost:5000/pages/home.html:276:16)
```

## AI 分析过程

1. 定位错误位置：`home.html` 第 276 行附近的 `renderSidebarCalories` 函数
2. 检查函数代码：`remaining.toLocaleString()` 中 `remaining` 为 `undefined`
3. 追溯数据来源：`calories` 变量来自 `API.diet.getTodayCalories()`
4. 测试后端 API：所有端点返回 200，数据正常
5. 分析 `request` 函数：`res.json()` 总是返回解析后的 JSON 对象
6. 发现关键问题：当 session 失效时 API 返回 `{success: false, message: '未登录'}`
7. 代码未检查响应格式，直接将错误对象当作正常数据使用

**结论：** 代码缺少对 API 错误响应的防御性检查。当 session 失效时，错误响应对象被当作正常数据传入渲染函数，导致访问不存在的属性时报错。

## 修复方案

在 `initPage` 函数中添加：
1. 未登录检测：`user.success === false` 时跳转登录页
2. 数据有效性检查：每个渲染函数前验证数据结构是否正确

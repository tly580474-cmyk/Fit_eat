# AI 询问提示日志：后台管理与登录问题排查

**日期：** 2026-05-23 19:12

## 用户反馈

用户报告两个问题：
1. 管理员后台的用户管理有 bug
2. 轻食一刻的账户无法正常登录

## AI 分析过程

### 问题 1：登录失败

1. 检查数据库中的用户数据：
   - 用户表 `users` 存在，`account_id` 字段已设置
   - admin 用户 `account_id='admin'`，密码验证通过
   - 测试用户 `account_id='linyouxue'`，密码验证通过

2. 测试登录 API：
   - `POST /api/auth/login` 使用 `accountId` 和 `password` 参数
   - 后端查询 `User.query.filter_by(account_id=account_id)`
   - 测试结果：API 正常工作

3. 检查前端登录页面 `login.html`：
   - 发现测试账号提示显示 "用户：轻食一刻 / 123456"
   - 但实际 `account_id` 是 "linyouxue"，不是 "轻食一刻"

**结论：** 用户按照页面提示输入 "轻食一刻" 作为账户ID，但系统中不存在该账户ID，导致登录失败。

### 问题 2：管理后台编辑报错

1. 测试管理后台页面：
   - `/admin/admin_user/` 返回 200，表格正常显示
   - `/admin/admin_user/edit/?id=1` 返回 500 错误

2. 查看错误日志：
   ```
   AttributeError: 'tuple' object has no attribute 'items'
   ```
   错误发生在 WTForms 的 `fields.py` 中。

3. 检查版本：
   - Flask-Admin: 1.6.1
   - WTForms: 3.2.2

4. 查阅资料发现：Flask-Admin 1.6.1 与 WTForms 3.2.2 不兼容，这是已知问题。

**结论：** 版本不兼容导致表单生成失败。

## 修复方案

1. 升级 Flask-Admin 到 2.2.0
2. 移除 `admin/__init__.py` 中不兼容的 `template_mode` 参数
3. 更新登录页面的测试账号提示
4. 在 `UserModelView` 中添加 `account_id` 列

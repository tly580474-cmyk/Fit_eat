# AI 询问提示日志：用户资料保存失败

**日期：** 2026-05-23 18:11

## 用户反馈

用户修改用户名和头像后，退出账号再登录时，头像和用户名不能正常保存。

## AI 分析过程

1. 检查后端用户路由 `routes/user.py`：`update_profile()` 函数正确处理 username、avatar 等字段，并调用 `db.session.commit()`
2. 检查用户模型 `models/user.py`：字段定义正确，`to_dict()` 方法返回完整用户数据
3. 测试数据库更新功能：直接修改数据库能正常保存，排除后端问题
4. 检查前端 `js/common.js`：
   - `EditProfileModal.submit()` 调用 `API.user.updateProfile(data)` 保存
   - 保存成功后调用 `initPage()` 重新加载数据
   - **关键发现**：`initPage()` 是异步函数，可能存在竞争条件
5. 检查 `pages/profile.html`：`initPage()` 从 API 获取数据并更新 DOM
6. **关键发现**：保存成功后没有立即更新页面显示，且 localStorage 中的用户名未同步更新

## 结论

属于前端刷新逻辑问题，后端数据库更新正常。问题在于：
1. 保存成功后调用 `initPage()` 存在异步竞争条件
2. 页面显示没有立即更新
3. localStorage 中的旧用户名未更新

## 修复方案

1. 添加 `updateProfileDisplay()` 函数统一更新页面显示
2. 保存成功后直接使用 API 返回的数据更新页面
3. 同步更新 localStorage 中的用户名

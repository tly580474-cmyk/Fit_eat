# AI 询问提示日志

**日期时间**: 2026-05-23 22:20

## 用户问题
在部分食谱的介绍当中，出现了类似 `![微波炉鸡蛋羹](./微波炉鸡蛋羹.jpg)` 的显示错误，这似乎是bug，请你检查每一个菜谱，并且修复类似的内容。

## 问题分析
1. **根本原因**: `import_howtocook.py` 中的 `parse_recipe` 函数在提取食谱描述时，没有清除 Markdown 图片语法 `![alt](url)`
2. **影响范围**: 数据库中 116 条食谱记录的 `description` 字段包含原始 Markdown 图片语法
3. **显示问题**: 前端使用 `textContent` 显示描述，导致 Markdown 语法原样显示为文本

## 涉及文件
- `轻饮食软件后端/import_howtocook.py` - 食谱导入脚本
- `instance/light_diet.db` - SQLite 数据库

## 解决方案
1. 修改 `import_howtocook.py` 的 `parse_recipe` 函数，添加正则表达式清除 Markdown 图片语法
2. 编写脚本清理数据库中已存在的 116 条包含 Markdown 图片语法的记录

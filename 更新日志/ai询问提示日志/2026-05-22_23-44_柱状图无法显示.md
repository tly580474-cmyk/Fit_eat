# AI 询问提示日志：摄入量柱状图无法显示

**日期：** 2026-05-22 23:44

## 用户反馈

记录页面的摄入量分析柱状图无法正常显示，区域空白。用户确认后说"柱子形状能显示，但高度不对"。

## AI 分析过程

### 第一轮（23:30）
1. 检查 `record.html` 的 `renderWeeklyChart` 函数：柱子高度由 `(item.consumed / maxValue) * 100` 计算
2. 发现无数据时 `maxValue=0`，高度为 0%
3. 添加最小高度 4%，但用户反馈仍不显示

### 第二轮（23:35）
1. 用 Playwright 截图确认：柱子元素存在但高度为 0
2. 分析 CSS：`h-full`（flex 子元素）+ 百分比 `height` 组合在浏览器中不生效
3. 重写为绝对定位 + 像素高度方案

### 第三轮（23:40）
1. 用户说"柱子形状能显示，但高度不对"
2. 查询数据库：`DietRecord` 表为空，API 返回 consumed 全为 0
3. 同时发现"本周平均: 1,840 kcal"是硬编码 HTML，不反映实际数据

## 结论

三层问题叠加：
1. CSS 布局：百分比高度在 flex 布局中不生效 → 改用绝对定位 + 像素
2. 数据缺失：`init_db.py` 没有创建 DietRecord 种子数据 → 添加 7 天记录
3. 摘要硬编码：统计文字写死在 HTML 中 → 改为动态 API 驱动

## 修复

1. `renderWeeklyChart` 函数完全重写
2. `init_db.py` 添加 DietRecord 和 WaterRecord 种子数据
3. 统计文字改为动态 `<span>` 由 API 数据填充

## 三、数据库设计（12 张表）

### 核心表结构

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `users` | 用户 | username, email, password_hash, gender, age, height, weight, target_calories |
| `foods` | 食物库 | name, calories, protein, carbs, fat, fiber, tags, ingredients(JSON), steps(JSON) |
| `diet_records` | 饮食记录 | user_id, food_id, meal_type, calories, protein, amount, recorded_at |
| `water_records` | 饮水记录 | user_id, amount, recorded_at |
| `community_posts` | 社区动态 | user_id, content, image, location, category |
| `comments` | 评论 | post_id, user_id, content |
| `likes` | 点赞 | post_id, user_id（唯一约束） |
| `follows` | 关注 | follower_id, following_id（唯一约束） |
| `achievements` | 成就定义 | name, icon, color, condition_type, condition_value |
| `user_achievements` | 用户成就 | user_id, achievement_id, unlocked_at |
| `ai_body_data` | AI身体数据 | gender, age, height, weight, bmi, daily_calories, protein, carbs, fat |

### 关系设计

- User 1:N DietRecord（用户有多条饮食记录）
- User 1:N WaterRecord（用户有多条饮水记录）
- User 1:N CommunityPost（用户发布多条动态）
- CommunityPost 1:N Comment（动态有多条评论）
- CommunityPost 1:N Like（动态有点赞）
- User M:N Achievement（通过 UserAchievement 关联）
- DietRecord N:1 Food（记录关联食物库）

## 六、前端对接方式

前端 `js/api.js` 已从 mock 数据切换为真实 fetch 请求：

```javascript
const API_BASE_URL = 'http://localhost:5000/api';

async function request(url, options = {}) {
  const token = localStorage.getItem('token');
  const headers = { 'Content-Type': 'application/json', ...options.headers };
  if (token) headers['Authorization'] = `Bearer ${token}`;
  const res = await fetch(`${API_BASE_URL}${url}`, { ...options, headers, credentials: 'include' });
  return res.json();
}
```

Flask 同时托管前端静态文件（HTML/CSS/JS），无需单独的前端服务器。

# Bug 修复日志

**日期：** 2026-05-23 19:12

## 问题描述

1. 管理后台用户管理页面无法编辑用户，点击编辑报 500 错误
2. 轻食一刻的测试账户无法正常登录
3. 管理后台用户列表缺少 `account_id` 列

## 问题分析

### Bug 1：管理后台编辑用户报 500 错误

**错误信息：**
```
AttributeError: 'tuple' object has no attribute 'items'
```

**根本原因：** Flask-Admin 1.6.1 与 WTForms 3.2.2 版本不兼容。新版 WTForms 修改了内部数据结构，导致 Flask-Admin 的表单生成代码出错。

**受影响文件：** `admin/__init__.py`

### Bug 2：测试账户无法登录

**根本原因：** 登录页面的测试账号提示显示 "轻食一刻"，但实际的 `account_id` 是 "linyouxue"。用户按照提示输入 "轻食一刻" 作为账户ID，自然无法登录。

**受影响文件：** `轻饮食软件前端/pages/login.html`

### Bug 3：用户列表缺少 account_id 列

**根本原因：** `UserModelView` 的 `column_list` 中没有包含 `account_id` 字段，导致管理后台看不到用户的登录账户ID。

**受影响文件：** `admin/views.py`

## 修复方案

### 修复 1：升级 Flask-Admin

```bash
pip install flask-admin --upgrade
```

将 Flask-Admin 从 1.6.1 升级到 2.2.0，并移除 `admin/__init__.py` 中不兼容的 `template_mode='bootstrap4'` 参数。

### 修复 2：更新测试账号提示

修改 `login.html` 中的测试账号提示：
```
用户：轻食一刻 / 123456
→
用户：linyouxue / 123456
```

### 修复 3：添加 account_id 列

修改 `admin/views.py` 中的 `UserModelView`，将 `account_id` 添加到 `column_list` 中，并添加对应的中文标签。

## 修改文件

| 文件 | 修改内容 |
|------|----------|
| `admin/__init__.py` | 移除 `template_mode` 参数 |
| `admin/views.py` | 添加 `account_id` 列和标签 |
| `轻饮食软件前端/pages/login.html` | 更新测试账号提示 |

## 验证结果

- 管理后台用户列表正常显示 `account_id` 列
- 管理后台编辑用户功能正常工作
- 测试账户 `linyouxue / 123456` 可以正常登录
- 管理员账户 `admin / admin123` 可以正常登录

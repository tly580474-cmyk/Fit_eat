# Bug 修复日志

**日期时间：** 2026-05-23 22:14

## Bug 描述

发现页面食谱图片全部无法显示，浏览器报 `ERR_ABORTED` 错误。175 张图片中 173 张是 Git LFS 指针文件（130 字节），不是真正的图片数据。

## 根本原因

HowToCook 仓库使用 Git LFS 存储图片文件。用户克隆/下载仓库时，LFS 文件未被正确拉取，只下载了 LFS 指针文件。`import_images.py` 直接复制这些指针文件到 `uploads/recipes/`，导致 Flask 提供的"图片"实际上是文本指针，浏览器无法渲染。

## 修复方案

### 重写 `import_images.py`

将原来的"从本地复制"改为"从 GitHub 下载"：

1. 通过 GitHub API (`api.github.com/repos/.../git/trees/master?recursive=1`) 获取所有图片文件路径
2. 使用 `media.githubusercontent.com/media/...` 下载 LFS 文件的实际内容（raw.githubusercontent.com 对 LFS 文件只返回指针）
3. 按菜品名分组，选择最佳图片（优先"成品"、"完成"等关键词）
4. 保存到 `uploads/recipes/` 并更新数据库

### 关键技术点

- Git LFS 文件的正确下载 URL：`https://media.githubusercontent.com/media/{owner}/{repo}/{branch}/{path}`
- 不是 `raw.githubusercontent.com`（该 URL 对 LFS 文件返回指针内容）

### 清理旧数据

- 删除 `uploads/recipes/` 下所有小于 1000 字节的损坏文件（173 个）
- 下载 174 张真实图片，总计 176 张可用图片

### 更新 `init_db.py`

`init_db.py` 已包含自动调用 `import_images()` 的逻辑，重置数据库后会自动从 GitHub 下载图片。

## 修改文件

| 文件 | 修改内容 |
|------|----------|
| `轻饮食软件后端/import_images.py` | 完全重写：从 GitHub API + media URL 下载图片 |

## 验证结果

- 176 张图片全部为真实图片数据（最小 28KB，最大 1MB）
- 数据库路径正确指向 `/api/uploads/recipes/{filename}.jpg`
- Flask 静态文件服务正常

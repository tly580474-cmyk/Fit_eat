# Bug 修复日志

**日期：** 2026-05-23 21:11

## 问题描述

首页 `home.html` 控制台报错：
```
TypeError: Cannot read properties of undefined (reading 'toLocaleString')
at initPage (http://localhost:5000/pages/home.html:276:16)
```

## 问题分析

**根本原因：** 当用户 session 失效或服务重启后，API 返回 `{success: false, message: '未登录'}` 错误响应。但 `initPage` 函数没有检查返回数据的有效性，直接将错误对象当作正常数据传入渲染函数。

**触发条件：**
- 服务重启后 session 丢失
- 用户长时间未操作导致 session 过期
- 浏览器保留了旧的 token 但服务端 session 已清除

**错误链路：**
1. `API.diet.getTodayCalories()` 返回 `{success: false, message: '未登录'}`
2. `calories` 变量被赋值为错误对象
3. `renderSidebarCalories(calories)` 尝试访问 `calories.target`
4. `calories.target` 为 `undefined`
5. `undefined.toLocaleString()` 抛出 TypeError

## 修复方案

在 `initPage` 函数中添加数据有效性检查：

1. 检测未登录响应，自动跳转登录页
2. 渲染前验证数据结构：
   - `calories.target !== undefined` 才调用热量渲染
   - `Array.isArray(meals)` 才调用餐次渲染
   - `macros.protein` 存在才调用营养素渲染
   - `Array.isArray(records)` 才调用记录渲染

## 修改文件

| 文件 | 修改内容 |
|------|----------|
| `轻饮食软件前端/pages/home.html` | initPage 添加数据有效性检查 |

## 验证结果

- 正常登录状态下首页数据正常显示
- session 失效时不再报错，自动跳转登录页
- 部分 API 失败时不影响其他正常数据的渲染

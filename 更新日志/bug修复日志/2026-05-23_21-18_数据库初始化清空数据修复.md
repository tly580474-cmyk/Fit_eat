# Bug 修复日志

**日期：** 2026-05-23 21:18

## 问题描述

每次按照启动步骤运行 `python init_db.py` 启动应用后，之前创建的社区帖子、饮食记录等用户数据全部消失。

## 问题分析

**根本原因：** `init_db.py` 中的 `init_database()` 函数每次执行都会调用 `db.drop_all()`，该操作会**删除数据库中所有表及数据**，然后 `db.create_all()` 重新创建空表，最后填充示例数据。

**问题代码（init_db.py 第 16 行）：**
```python
def init_database():
    with app.app_context():
        db.drop_all()      # 删除所有表和数据
        db.create_all()    # 重建空表
        # ... 填充示例数据
```

**触发条件：**
- 用户每次启动应用都按照启动步骤运行 `python init_db.py`
- 启动步骤中虽标注"首次运行"，但未做防护，重复执行会直接清空数据

## 修复方案

1. 在 `init_database()` 中添加数据库存在性检查
2. 数据库已存在时自动跳过初始化，不清空数据
3. 新增 `--force` 参数用于强制重置数据库（仅在需要时使用）

**修复后逻辑：**
```python
def init_database(force=False):
    import os
    db_path = os.path.join(...)

    if os.path.exists(db_path) and not force:
        print('数据库已存在，跳过初始化')
        return

    db.drop_all()
    db.create_all()
    # ... 填充示例数据
```

**使用方式：**
- `python init_db.py` — 数据库已存在时自动跳过，安全无损
- `python init_db.py --force` — 强制清空重建（慎用）

## 修改文件

| 文件 | 修改内容 |
|------|----------|
| `轻饮食软件后端/init_db.py` | 添加数据库存在性检查和 --force 参数 |

## 验证结果

- 运行 `python init_db.py` 时数据库已存在，输出"跳过初始化"
- 跳过初始化后社区帖子数据（8条）完好保留
- 运行 `python init_db.py --force` 可正常清空重建数据库

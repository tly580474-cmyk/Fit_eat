# Bug 修复日志：用户资料保存失败

**日期：** 2026-05-23 18:11

## 问题描述

用户修改用户名和头像后，退出账号再登录时，头像和用户名不能正常保存。

## 问题分析

经过代码审查和测试，发现以下问题：

1. **后端数据库更新正常**：`update_profile` 函数能正确保存数据到数据库
2. **前端刷新逻辑问题**：保存成功后调用 `initPage()` 重新加载数据，存在异步竞争条件
3. **页面显示未同步**：保存成功后没有立即更新页面上的用户信息显示
4. **localStorage 未同步**：修改用户名后，localStorage 中的旧用户名未更新

## 修复方案

### 1. 添加 `updateProfileDisplay()` 函数

在 `profile.html` 中添加统一的用户数据更新函数，用于更新页面上的所有用户信息显示。

### 2. 优化保存逻辑

在 `EditProfileModal` 和 `AvatarModal` 中，保存成功后直接使用 API 返回的数据调用 `updateProfileDisplay()` 更新页面，而不是重新调用 `initPage()`。

### 3. 同步 localStorage

在 `updateProfileDisplay()` 函数中，同时更新 localStorage 中的用户名，确保数据一致性。

## 修改文件

- `轻饮食软件前端/js/common.js` — 优化 `EditProfileModal.submit()` 和 `AvatarModal` 的保存逻辑
- `轻饮食软件前端/pages/profile.html` — 添加 `updateProfileDisplay()` 函数，优化页面初始化

## 技术实现

1. `updateProfileDisplay(user)` 函数：接收用户数据对象，更新页面上的头像、用户名、简介等显示
2. 保存成功后，使用 `res.data`（API 返回的最新用户数据）直接更新页面
3. 同时更新 `localStorage.setItem('username', user.name)` 确保数据同步

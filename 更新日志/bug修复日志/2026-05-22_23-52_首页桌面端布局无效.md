# Bug 修复日志：首页桌面端布局无效

**日期：** 2026-05-22 23:52

## 问题描述

电脑端（宽度 >= 1024px）打开首页，页面仍按手机版单列排版，没有显示桌面端双栏布局（左侧边栏 + 右侧主内容）。

## 原因分析

首页使用自定义 CSS 类（`desktop-grid-12`、`desktop-sidebar`、`desktop-main`）实现桌面端 12 栏网格布局，但这些类与 Tailwind CDN 的样式系统存在兼容性问题：

1. **自定义 CSS 与 Tailwind 冲突** — `desktop-grid-12` 的 `display: grid` 可能被 Tailwind 的 reset 样式覆盖
2. **响应式断点不一致** — 自定义 CSS 用 `min-width: 1024px`（lg），但 Tailwind 的 `lg:` 前缀有自己的优先级机制
3. **移动端/桌面端内容重复** — 侧边栏内容（热量、宏量营养素）在移动端和桌面端都显示，未做可见性区分

## 修复方案

将自定义 CSS 类替换为 Tailwind 原生响应式类：

1. **网格容器**: `desktop-grid-12` → `grid grid-cols-1 gap-8 lg:grid-cols-12`
2. **侧边栏**: `desktop-sidebar` → `hidden lg:flex lg:flex-col lg:gap-6 lg:col-span-4 lg:sticky lg:top-24 lg:self-start`
3. **主内容**: `desktop-main` → `flex flex-col gap-6 lg:col-span-8`
4. **食谱卡片网格**: 桌面端 `lg:grid lg:grid-cols-3` 替代横向滚动
5. **饮食记录网格**: 桌面端 `lg:grid lg:grid-cols-2` 替代单列
6. **MealCard 组件**: 添加 `lg:min-w-0` 覆盖 `min-w-[260px]`，确保网格布局生效
7. **底部导航栏**: 添加 `lg:hidden` 双重保险

## 修改文件

- `轻饮食软件前端/pages/home.html` — 网格布局、侧边栏、主内容区、食谱卡片、饮食记录
- `轻饮食软件前端/js/components.js` — MealCard 添加 `lg:min-w-0`

## 验证步骤

1. 启动服务 `cd 轻饮食软件后端 && python app.py`
2. 浏览器打开 `http://localhost:5000/pages/home.html`
3. 浏览器窗口宽度 >= 1024px 时，应显示左侧 4 栏边栏 + 右侧 8 栏主内容
4. 窗口宽度 < 768px 时，应恢复单列移动布局

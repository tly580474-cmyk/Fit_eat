# Bug 修复日志：摄入量柱状图无法显示

**日期：** 2026-05-22 23:44

## 问题描述

记录页面的"摄入量分析"柱状图区域空白，柱子不可见。即使 API 返回了热量数据，柱状图依然不显示。

## 原因分析

**第一层：CSS 布局问题（23:30）** — Flex 布局中 `h-full` 与百分比 `height` 组合存在兼容性问题。柱子使用 `style="height: ${height}%"` 但父容器的高度链断裂，导致柱子实际高度为 0。

**第二层：数据库无数据（23:40）** — 柱状图虽然渲染了，但 `init_db.py` 中没有创建 `DietRecord` 数据，API `/api/diet/weekly-calories` 返回的 `consumed` 全为 0，柱子高度为 0px。

**第三层：摘要文字硬编码（23:40）** — "本周平均: 1,840 kcal" 和 "12% 目标达成" 是写死在 HTML 中的，不反映实际数据。

## 修复方案

### 1. CSS 布局重写（前端 `record.html`）
改用**绝对定位 + 像素高度**方案：
- 根据 `container.offsetHeight` 获取容器实际像素高度
- 柱子使用 `position:absolute; bottom:0` 定位
- 高度用像素值 `height:${barHeight}px` 代替百分比
- X 轴标签独立渲染在底部

### 2. 添加种子数据（后端 `init_db.py`）
为测试用户"林小悦"添加 7 天的饮食记录和饮水记录：
- 7 天饮食记录：每天 2-3 餐，使用现有食谱数据
- 7 天饮水记录：每天 6 杯 × 250ml
- 需要用户重新运行 `python init_db.py`

### 3. 动态摘要文字（前端 `record.html`）
- "本周平均" 改为 `<span id="weekly-avg">`，由 API 数据填充
- "目标达成" 改为 `<span id="weekly-progress">`，由 API 数据计算

## 修改文件

- `轻饮食软件前端/pages/record.html` — `renderWeeklyChart` 函数重写 + 动态摘要
- `轻饮食软件后端/init_db.py` — 添加 DietRecord、WaterRecord 种子数据

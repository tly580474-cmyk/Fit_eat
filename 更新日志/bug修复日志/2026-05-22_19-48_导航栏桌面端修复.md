## Bug 描述

桌面端浏览器（宽度 >= 768px）打开页面后，底部导航栏被隐藏，顶部导航栏没有页面跳转链接，导致用户无法在页面间切换。

## 根本原因

底部导航栏使用 `mobile-only` 类，CSS 媒体查询在桌面端将其隐藏：
```css
@media (min-width: 768px) {
  .mobile-only {
    display: none !important;
  }
}
```
但顶部导航栏未提供桌面端导航替代方案。

## 修复方案

在每个页面的顶部导航栏中添加桌面端导航链接，使用 `hidden md:flex` 控制显隐：

```html
<nav class="hidden md:flex items-center gap-6 ml-8">
  <a href="home.html">首页</a>
  <a href="discover.html">发现</a>
  <a href="record.html">记录</a>
  <a href="discover.html">社区</a>
  <a href="profile.html">我的</a>
</nav>
```

当前页面链接使用 `text-green-700 font-bold` 高亮，其余页面使用 `text-gray-600`。

- 桌面端（>= 768px）：顶部显示文字导航链接，底部导航栏隐藏
- 移动端（< 768px）：顶部不显示文字导航，底部显示图标导航栏
